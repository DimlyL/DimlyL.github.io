ODT 学习笔记

Posted on 2021-12-31 Edited on 2022-06-03 In 学习笔记 Views:

是 Dimly 的 ODT 学习笔记耶！

老司机树，ODT(Old Driver Tree)，又名珂朵莉树(Chtholly Tree)。起源自 CF896C。

是一种基于 set 的暴力数据结构，在随机数据且有区间赋值操作时有较为优秀的效率。

结构体变量。

l：该节点代表的区间的左端点。

r：该节点代表的区间的右端点。

v：该节点代表的区间里所有数的值。

bool operator <：将节点在 set 中按 l 递增排序。

将每个节点单独插入。当然，如果每个数初始时都一样，可以直接整体插入。

因为 split 的操作，要额外加一个无实际意义节点以防止 RE。

Upd：不过发现好像删掉这玩意好像也没啥事……我再仔细思考一下。

宏定义 IT 为 set<node> 的迭代器。

ODT 的核心操作。将节点分裂开后方便处理。

将包含区间内 的节点（设为 ）分裂为两个区间 和 并返回指向后者的迭代器。

ODT 的核心操作，区间赋值。

将区间 分裂出来后删除并重新插入。

（关键点：一定要先分裂 R 再分裂 L，若先分裂 L 再分裂 R 则有可能因为 R 的分裂导致 L 的迭代器失效）

区间加。将区间 分裂出来后暴力加。

区间和。将区间 分裂出来后暴力求和。

交换两个区间，可以用两个队列来辅助实现。

注意在对 ODT 的结构进行修改后（insert 和 erase）都要重新用 split 获取迭代器。
